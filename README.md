# Lost Minions Games — Organization Hub

Welcome to **Lost Minions Games**, the game-focused branch of the Lost Minions ecosystem.
This repository serves as the **central configuration and community home** for all projects under the `LostMinionsGames` organization, including the experimental **Glitch** project.

---

## About the Project

Lost Minions Games is where prototypes, experiments, and full game projects live.
What started as a small cluster of tools and ideas has grown into an organized space for:

> _Gameplay experiments and prototypes._
> _Shared build and CI workflows for Unreal Engine and supporting services._
> _Org-wide labels, templates, and automation tuned for game dev._

Every repository in **LostMinionsGames** connects back to this shared configuration repo for community standards, automation, and contributor experience.

## Issue Templates

All repositories in **LostMinionsGames** share unified issue templates so contributors can report bugs, propose features, or request design updates consistently.

| Type | Purpose |
|------|---------|
| **Bug Report** | Something is broken or behaving unexpectedly in a game or workflow. |
| **Feature Request** | Suggest new gameplay features, tools, or automation. |
| **Task** | General maintenance, refactors, or pipeline work. |
| **Documentation** | Update READMEs, contributor guides, or internal docs. |
| **Design / UX** | Propose or refine UX, UI, or visual elements for games. |

These templates live here and are synced out to game repos so the experience is consistent across the org.

---

## Shared Standards

Across all **LostMinionsGames** projects we maintain a unified label and metadata system:

- **Game areas:** `gameplay`, `ux-ui`, `content`, `tools`
- **Maintenance:** `task`, `enhancement`, `refactor`, `documentation`
- **Standard GitHub:** `bug`, `help wanted`, `question`, `good first issue`, etc.

This shared vocabulary keeps work organized across code, design, and infrastructure.
